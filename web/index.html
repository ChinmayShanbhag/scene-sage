How It Works

End-to-End Pipeline

From raw YouTube trailers to a searchable multimodal index in five stages.

01

Ingest Trailers

Download MP4 trailers and English subtitle files from YouTube using yt-dlp with automated batch processing.

yt-dlp · PowerShell
02

Extract Frames

Extract one frame per second from each trailer using FFmpeg, with an OpenCV fallback for robustness.

FFmpeg · OpenCV
03

Align Captions

Parse SRT subtitles and match each frame to its nearest-in-time caption using temporal alignment.

SRT Parser · Temporal Match
04

Encode Vectors

Encode frames with OpenCLIP (ViT-B-32) and captions with Sentence Transformers, then concatenate into 896-dim vectors.

OpenCLIP · MiniLM
05

Index & Search

Build a FAISS inner-product index over L2-normalized vectors for sub-millisecond cosine similarity search.

FAISS · Cosine Similarity

Technology

Built With

A carefully chosen stack combining state-of-the-art vision-language models with efficient vector search.

OpenCLIP

Vision Transformer (ViT-B-32) pretrained on OpenAI's CLIP dataset. Encodes each frame into a 512-dimensional visual embedding.

ViT-B-32 512-dim FP16 CUDA

Sentence Transformers

MiniLM-L6-v2 model encodes aligned subtitle text into 384-dimensional semantic embeddings, capturing dialogue context.

all-MiniLM-L6-v2 384-dim Semantic

FAISS

Facebook AI Similarity Search builds an inner-product index over L2-normalized vectors for blazing-fast cosine similarity lookups.

IndexFlatIP Cosine Sim 50K+ vectors

FFmpeg + OpenCV

Dual extraction strategy: FFmpeg as primary (fast, accurate fps filter) with OpenCV frame-stepping as a robust fallback.

1 fps JPG output Fallback

PyTorch

GPU-accelerated tensor operations with CUDA support for efficient batch encoding of thousands of frames.

CUDA DataLoader FP16

yt-dlp

Automated YouTube trailer ingestion with subtitle extraction, format selection, and batch downloading from ID lists.

MP4 SRT Batch
Under the Hood

Core Encoding Logic

Each frame is encoded into a fused vision-language vector that captures both what's seen and what's said.

scene_sage_baseline.ipynb — encode & index 
# Encode frames with OpenCLIP (visual) + Sentence Transformers (text)
with torch.no_grad():
    v_img = clip_model.encode_image(imgs).cpu().numpy()
    v_txt = st_model.encode(captions)

# Concatenate: 512-dim image + 384-dim text = 896-dim multimodal
v_full = np.hstack([v_img, v_txt])

# L2-normalize for cosine similarity via dot product
faiss.normalize_L2(vec_mat)

# Build FAISS inner-product index
index = faiss.IndexFlatIP(896)
index.add(vec_mat)

# Search: find k most similar frames
D, I = index.search(query_vec, k=50)

Capabilities

Key Features

Designed for robustness, scalability, and meaningful multimodal understanding.

Multimodal Fusion

Combines visual features from OpenCLIP with semantic text embeddings from Sentence Transformers into a unified 896-dimensional representation.

Sub-millisecond Search

FAISS inner-product index enables near-instant cosine similarity search across 50K+ frame vectors with trailer-level deduplication.

Robust Extraction

Dual-strategy frame extraction with FFmpeg as primary and OpenCV as fallback, ensuring reliable processing across video formats.

Caption-Aware

Temporal alignment of SRT subtitles to frames adds dialogue context, making search sensitive to both visual content and spoken words.

Trailer Diversity

Recommendation logic deduplicates at the trailer level, ensuring search results surface frames from distinct, diverse trailers.

Batch Processing

GPU-accelerated batch encoding with PyTorch DataLoader processes hundreds of trailers and tens of thousands of frames efficiently.
